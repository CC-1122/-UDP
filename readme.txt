UDP Socket 编程实验作业
========================

本作业实现了一个基于UDP的可靠数据传输协议，模拟TCP的连接建立、超时重传和滑动窗口流量控制机制。

文件列表
--------
1. udpserver.py - 服务端程序
2. udpclient.py - 客户端程序
3. readme.txt - 程序运行说明文档

运行环境
--------
- Python 3.x
- 无需额外安装第三方库（仅使用标准库）

运行方式
--------

1. 启动服务端：
   python udpserver.py <端口号>
   
   示例：
   python udpserver.py 12345

2. 启动客户端：
   python udpclient.py <服务器IP> <服务器端口> <学号后4位>
   
   示例：
   python udpclient.py 127.0.0.1 12345 2906

协议规范
--------

数据包格式：
- 魔数：2字节 (0xABCD)
- 数据包类型：1字节
  - 0x01: 连接请求
  - 0x02: 连接确认
  - 0x03: 数据报文
  - 0x04: 数据确认

连接请求 (0x01):
- 魔数 (2字节)
- 数据包类型 (1字节)
- StudentID (2字节)

连接确认 (0x02):
- 魔数 (2字节)
- 数据包类型 (1字节)
- 成功标志 (1字节: 1=成功, 0=失败)

数据报文 (0x03):
- 魔数 (2字节)
- 数据包类型 (1字节)
- 序列号 (1字节)
- 起始字节 (2字节)
- 结束字节 (2字节)
- 数据载荷 (可变长度)

数据确认 (0x04):
- 魔数 (2字节)
- 数据包类型 (1字节)
- 序列号 (1字节)
- 服务器小时 (1字节)
- 服务器分钟 (1字节)
- 服务器秒 (1字节)

StudentID 验证机制
------------------
1. 取学号后4位数字
2. 与 0x5A3C 进行 XOR 运算
3. 将结果填入连接请求报文中
4. 服务器收到请求后，对收到的值再次 XOR 0x5A3C
5. 检查结果是否在 0-9999 范围内

实现功能
--------
1. 基于UDP的连接建立（带StudentID验证）
2. 服务端随机丢包模拟（丢包率20%）
3. 客户端超时重传（超时时间300ms）
4. 滑动窗口流量控制（窗口大小5）
5. RTT计算与统计
6. 运行日志记录（run_log.txt）
7. 汇总统计信息输出

程序输出
--------

服务端日志 (run_log.txt):
- 时间戳
- 连接建立/拒绝
- 数据包接收/发送
- 丢包记录

客户端控制台:
- 连接状态
- 数据包发送状态
- ACK接收与RTT
- 重传通知
- 汇总统计信息

汇总统计信息
------------
发送30个数据包后，客户端显示：
- 丢包率百分比
- 最大RTT
- 最小RTT
- 平均RTT
- RTT标准差
- 实际发送的UDP包数
- 成功确认的包数
- 重传的包数

注意事项
--------
- 服务端运行后需按 Ctrl+C 终止
- 客户端默认发送30个数据包
- 丢包率设置为20%
- 超时时间设置为300ms
- 窗口大小设置为5个数据包
